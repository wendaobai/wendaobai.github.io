可执行上线下文 (execution context) #

什么是可执行上下文 #

执行上下文是评估和执行 JavaScript 代码的环境的抽象概念。每当 Javascript 代码在运行的时候，它都是在执行上下文中运行。简单来说就是代码运行时的环境，包括了变量作用域，作用域链，this指向

可执行上下文会创建一个变量对象（variable object VO）

基础数据类型都保存在变量对象中

引用类型的值保存在堆内存中，我们通过操作引用堆内存地址来操作该对象

javaScript
function Fn(){
    var a = 1;
    var b = {
        name:'zhufeng'
    }
    var c = [1,2,3];
}
let ExecuteContext = {
    VO:{
        a:1,
        b:'XO1',
        c:'XA1'
    }
}

怎么创建可执行上下文 #

  1. 全局可执行上下文，不会回收
    • 全局可执行上下文只有一个，在客户端一般由浏览器创建，就是我们熟悉的window对象，我们通过this访问到他
    • 通过var声明的全局变量也会挂载在globalExecutionContextVO下，或者没有通过var let const声明的变量也会挂载在window下
  2. 函数可执行上下文，函数执行完成，出栈时垃圾回收
  3. eval可执行上下文

多个可执行上下文（执行栈） #

  • 在JS执行过程会产出多个执行上下文,JS引擎会有栈来管理这些执行上下文
  • 执行上下文栈(下文简称执行栈)也叫调用栈，执行栈用于存储代码执行期间创建的所有上下文，具有LIFO（Last In First Out后进先出，也就是先进后出）的特性
  • 栈底永远是全局上下文，栈顶为当前正在执行的上下文
  • 当开启一个函数执行时会生成一个新的执行上下文并放入调用栈，执行完毕后会自动出栈

WARNING

JavaScript是静态作用域，在函数定义时就确定了它的作用域，而不是运行时才确认（就近原则）

在创建VO对象挂载是函数，就创建该函数的可执行上下文，vo对象等等

可执行上下文的生命周期 #

执行上下文分两个阶段

  • 创建阶段
    • 创建变量对象VO
    • 确定作用域链scopeChain
    • 确定this指向
  • 执行阶段
    • 变量赋值
    • 代码执行

变量对象VO #

变量对象会保存变量声明（var声明的变量）、函数参数（arguments）、函数声明（function）

  • VO对象会优先处理function声明的函数，获取函数的参数与值（函数也是对象，即引用堆内存地址）

    函数名为Vo的属性名，值为函数对象，如果属性已经存在，则用新值覆盖

  • Vo对象然后处理var声明变量

    每一个var变量声明，就会在变量对象上建一个属性,然后初始化undefined,如果变量名已经存在，则会跳过，并不会修改原属性值,let声明的变量并不会在此阶段进行处理

    这就是var的变量提升

  • 函数声明优先级更高，同名的函数会覆盖函数和变量，但同名var变量并不会覆盖函数.执行阶段重新赋值可以改变原有的值

console.log(a);

// 执行过程，编译阶段时会变量提升
var a  = undefined
console.log(a)// undefined
a = 1;

活动对象AO（运行时） #

  • 在函数的调用栈中，如果当前执行上下文处于函数调用栈的顶端，则意味着当前上下文处于激活状态，此时变量对象称为活动对象(AO,Activation Object) VO=>AO
  • 活动变量包含变量对象所有的属性，并有包含this指针,所以this是运行时才能确定
javaScript
function one(m) {
    function two() {
        console.log('two');
    }
}
one(1);
// 创建时
let VO =  {
    m:1,
    two: () => { console.log('two'); },//此时two的作用域也确认了，创建了twoExecutionContext

}
let oneEC={
    VO,
    this: null,
    scopeChain:[VO,globalVO]
}
//执行阶段 VO=>AO
AO = VO 
let oneEC={
    AO,
    this: window,
    scopeChain:[VO,globalVO]
}

全局上下文的变量对象 GlobalVo #

  • 在浏览器里，全局对象为window
  • 全局上下文的变量对象为window,而且这个变量对象不能激活变成活动对象
  • 只在窗口打开，全局上下文会一直存在，所有的上下文都可以直接访问全局上下文变量对象上的属性
  • 只有全局上下文的变量对象允许通过VO的属性名称来间接访问，在函数上下文中是不能直接访问VO对象的
  • 未进入执行阶段前，变量对象中的属性都不能访问！但是进入到执行阶段之后，变量对象转变成了活动对象，里面的属性都能被访问了,对于函数上下文来讲，活动对象与变量对象其实都是同一个对象,只是处于执行上下文的不同生命周期

作用域与作用域链 #

作用域 #

作用域指的是变量与函数的可以访问的范围，在编译阶段就确认了，所以是静态作用域

  • 全局作用域

    global scope程序全局声明的变量与函数、window挂载的属性与方法、

  • 局部作用域

    也就是函数作用域，

  • 块级作用域

javaScript
function one() {
    var a = 1;
}
console.log(a);// undefinded

作用域链 #

  • 作用域链是由当前执行环境与上层执行环境的一系列变量对象组成的，它保证了当前执行环境对符合访问权限的变量和函数的有序访问
javaScript
function one() {
    var a = 1;
    function two() {
        var b = 2;
        function three() {
            var c = 3;
            console.log(a, b, c);
        }
        three();
    }
    two();
}
one(); 
// 1.创建全局上下文
var globalExecuteContextVO = { one: `function()=>{var a = 1;}` }
var globalExecuteContext = {
    VO: globalExecuteContextVO,
    scopeChain: [globalExecuteContextVO]
}
var executeContextStack = [globalExecuteContext];
//2.执行one，创建one执行上下文
var oneExecuteContextVO = {
    a: 1,
    two: `()=>{var b = 2 ;}`
}
var oneExecuteContext = {
    VO: oneExecuteContextVO,
    scopeChain: [oneExecuteContextVO, globalExecuteContext.VO]
}
//2.执行two，创建two执行上下文
var twoExecuteContextVO = {
    b: 2,
    three: `()=>{var c = 3 ;}`
}
var twoExecuteContext = {
    VO: twoExecuteContextVO,
    scopeChain: [twoExecuteContextVO, oneExecuteContext.VO, globalExecuteContext.VO]
}
//3.执行three，创建three执行上下文
var threeExecuteContextVO = {
    c: 3
}
var threeExecuteContext = {
    VO: threeExecuteContextVO,
    scopeChain: [threeExecuteContextVO, twoExecuteContext.VO, oneExecuteContext.VO, globalExecuteContext.VO]
}
//变量查找规则，在作用域链上找，找到就返回
function getValue(varName) {
    for (let i = 0; i < threeExecuteContext.scopeChain.length; i++) {
        if (varName in threeExecuteContext.scopeChain[i]) {
            return threeExecuteContext.scopeChain[i][varName];
        }
    }
}

    getValue('a'),
    getValue('b'),
    getValue('c'),

this 指向 #

  • 当前函数的this是在被调用的时候才能确定的
  • 只有执行上下文处于栈顶，VO转化AO时，this的指向才能确定

调用方式 #

  1. 全局调用

    全局对象this指向本身,浏览器指向window

    javaScript
    var a=1;//声明绑定变量对象，但在全局环境中，变量对象就是全局对象
this.b=2;//this绑定全局对象
c=3;//赋值操作 隐式绑定

  2. 函数的点调用

    在一个函数上下文中，this由函数的调用者提供，由调用函数的方式来决定指向

    如果是函数执行,如果前面有点，那么点前面是谁this就是谁，即谁调用this指向谁

    javaScript
    let obj = {
    getName(){
        console.log(this);
    }
};
obj.getName();// 此时this指向 obj

  3. 函数的直接调用

    this就是window(严格模式下是undefined),自执行函数中的this一般都是window

    javaScript
    let obj = {
    getName(){
        console.log(this);
    }
};
let getName = obj.getName;
getName();// window

  4. 绑定事件

    给dom元素绑定事件，this一般指向dom元素本身

    javaScript
    dom.addEventListener('click',function(){
    console.log(this);// dom
})

  5. 箭头函数

    • 箭头函数没有自己的this
    • 也没有prototype
    • 也没有arguments
    • 无法创建箭头函数的实例

    由于箭头函数使得this从“动态”变成“静态”，下面两个场合不应该使用箭头函数。

    箭头函数体内的this对象，就是定义该函数时所在的作用域指向的对象，而不是使用时所在的作用域指向的对象。

    javaScript
    let fn = () => {
    console.log(this);
    console.log(arguments);//Uncaught ReferenceError: arguments is not defined
}
console.log(fn.prototype);//undefined
fn();//
new fn();//VM4416:8 Uncaught TypeError: fn is not a constructor

  6. 构造函数

    构造函数的this指向实例本身

    javaScript
    function Fn(){
	console.log(this)// obj
}
let obj = new Fn();

如何改变this指向 #

  1. 通过new实例化

    this此时指向的实例化的对象

  2. call,apply,bind

    第一个参数是改变this指向(非严格模式下，传递null/undefined指向也是window)

    call参数是依次传递，apply是以数组的方式传递

    javaScript
    !function (proto) {
    function getContext(context) {
        context = context || window;
        var type = typeof context;
        if (['number', 'string', 'boolean', 'null'].includes(type)) {
            context = new context.constructor(context);
        }
        return context;
    }
    function call(context, ...args) {
        context = getContext(context);
        const _fn = Symbol()
        context[_fn] = this;
        let result = context[_fn](...args);
        delete context[_fn];
        return result;
    }
    function apply(context, args) {
        context = getContext(context);
        const _fn = Symbol()
        context[_fn] = this;
        let result = context[_fn](...args);
        delete context[_fn];
        return result;
    }

    function bind(context, ...bindArgs) {
        return (...args) => this.call(context, ...bindArgs, ...args);
    }
    proto.call = call;
    proto.apply = apply;
    proto.bind = bind;
}(Function.prototype)

es6的词法环境 #

es6引入了块级作用域，由let、const定义，随之引入词法环境来构建可执行上下文

let的原理 #

var 定义变量放变量环境中

let const 定义的变量放在词法环境中，词法环境是一个栈型机构

javaScript
function fn() {
    var a = 1;
    let b = 2;
    {
        let b = 3;
        var c = 4;
        let d = 5;
        console.log(a, b, c, d);//1 3 4 5
    }
    {
        let b = 6;
        let d = 7;
        console.log(a, b, c, d);// 1 6 4 7
    }
}
fn();

/**
 * 1.global编译阶段
 */

let globalEC = {
    this: globalThis,
    outer: null,//外部的执行上下文,词法作用域就是静态作用域，就是指作用域是由代码中函数声明的位置来决定的
    variableEnvironment: {
        fn() { console.log(a, b, c, d) }
    },
    lexicalEnvironment: {}
}


/**
 * 2.fn编译阶段
 */
let fnEC = {
    this: globalThis,
    outer: globalEC,
    variableEnvironment: { a: undefined, c: undefined },
    lexicalEnvironment: [{ b: undefined }]//栈型结构
}
/**
 * 3.编译代码块1
 */
fnEC.variableEnvironment.a = 1;
fnEC.lexicalEnvironment.b = 2;
fnEC.lexicalEnvironment.push({
    b: undefined,
    d: undefined
});
/**
 * 4.执行代码块1
 */

fnEC.lexicalEnvironment[1].b = 3;
fnEC.variableEnvironment.c = 4;
fnEC.lexicalEnvironment[1].d = 5;

console.log(getValue('a', fnEC), getValue('b', fnEC), getValue('c', fnEC), getValue('d', fnEC));
function getValue(name, ec) {
    for (let i = ec.lexicalEnvironment.length - 1; i >= 0; i--) {
        if (name in ec.lexicalEnvironment[i]) {
            return ec.lexicalEnvironment[i][name];
        }
    }
    let currentVariableEnvironment = ec.variableEnvironment;
    while (currentVariableEnvironment) {
        if (name in currentVariableEnvironment) {
            return currentVariableEnvironment[name];
        }
        currentVariableEnvironment = currentVariableEnvironment.outer;
    }
    return null;
}

/**
 * 5.编译代码块2

fnEC.lexicalEnvironment.pop();
fnEC.lexicalEnvironment.push({
    b: undefined,
    d: undefined
});
/**
 * 5.执行代码块2

fnEC.lexicalEnvironment[1].b = 6;
fnEC.lexicalEnvironment[1].d = 7;
